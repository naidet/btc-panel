## Phase 3 🆕 HMM 市场状态识别

**文件**: `hmm_state.py`

**参考**: [MQL5: 隐马尔可夫模型在交易系统中的应用](https://www.mql5.com/zh/articles/17917)

**实现步骤**:

```
Step 3.1: 特征提取
  - 移动窗口波动率 (5/35/65/95/125/155/185/215/245/275 period std)
  - 价格动量 (ROC)
  - 布林带宽度 (震荡判定)

Step 3.2: K-means 预聚类
  - n_clusters = 5
  - 标准化的特征数据
  - 计算先验均值、协方差、转移矩阵

Step 3.3: GaussianHMM 训练
  - n_components = 3~5
  - 用 K-means 结果初始化先验
  - n_iter = 100

Step 3.4: 状态映射
  - HMM 隐藏状态 → 市场体制
  - 状态0: 强趋势上涨
  - 状态1: 强趋势下跌
  - 状态2: 震荡整理
  - 状态3: 过渡/转折
```

**验证标准**:
- [ ] HMM 状态切换频率合理 (不会每根K线换一次)
- [ ] 各状态的收益分布差异明显
- [ ] 状态与 ADX 趋势/震荡分类基本一致

## Phase 4 🆕 HMM 驱动策略自动切换

**文件**: `unified_hmm.py`

```
          ┌──────────────────────────┐
          │     HMM 市场状态识别器     │
          │  每4h 更新一次状态判断      │
          └──────────┬───────────────┘
                     │
        ┌────────────┼────────────┐
        ▼            ▼            ▼
  状态0/1:趋势   状态2:震荡    状态3:过渡
  ┌──────────┐ ┌──────────┐ ┌──────────┐
  │共振趋势   │ │震荡套利   │ │ 观望/    │
  │策略       │ │策略       │ │ 轻仓     │
  │80%资金   │ │80%资金   │ │20%资金   │
  └──────────┘ └──────────┘ └──────────┘
        │            │            │
        └────────────┼────────────┘
                     ▼
          ┌──────────────────────┐
          │   统一风控层          │
          │   日亏上限 / 回撤熔断  │
          │   共享资金池           │
          └──────────────────────┘
```

**核心机制**:
- 每4小时通过HMM判断当前市场状态
- 趋势状态: 共振策略主导 (追随趋势方向)
- 震荡状态: 套利策略主导 (均值回归)
- 过渡状态: 减仓或观望
- 统一止损保护 + 日亏熔断
