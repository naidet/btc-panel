# BTC 策略机器人 — 最终报告

## 🔧 已修复的 Bug

### 1. 数据时间顺序错误 (最严重的bug)
**问题**: `fetch_bars()` 返回的数据是乱序的 — 旧的`insert(0, bar)`逻辑加上错误的`end_time`翻页，导致数据既有重复又有乱序。回测相当于"从未来往过去跑"，结果完全无效。

**修复**: 
- 翻页参数改为 `end_time = oldest_bar - 1ms`（防止重复）
- 去重+按时间正序排序（旧→新）
- 所有历史回测结果已全部更新

### 2. PnL 计算被放大100倍
**问题**: 公式残留 `* 100` 乘数（从外汇代码遗留），导致 BTC 回测盈亏被放大100倍。$10k → $510k 实际上是 $10k → ~$10.5k。

**修复**: 移除 `* 100` 乘数

### 3. 手数计算公式错误
**问题**: 公式有 `* 0.0001` 汇率转换（外汇遗留），BTC不需要

**修复**: 改为 `lots = risk / sl_pts`

### 4. BTC价格显示错误
**问题**: 仪表盘显示的是最后4h K线的收盘价，可能过时1-4小时

**修复**: 增加 `real_price` 字段（从 Binance ticker API 实时获取），仪表盘优先显示实时价格

### 5. ADX计算越界崩溃
**问题**: 当数据量 `15 ≤ n < 28` 时，`calc_adx()` 会在 `adx[27]` 处越界

**修复**: 早期返回条件改为 `n < period * 2 (28)`

## 🏆 最佳方案 (经过参数优化)

### RSI+EMA 日线 — 无追踪止盈 (策略反转出场)

| 参数 | 值 |
|------|-----|
| 时间框架 | 日线 (1d) |
| 入场规则 | 价格>EMA20 + RSI 50-70(多) / 价格<EMA20 + RSI 30-50(空) |
| 出场规则 | 策略信号反转 / SL=1.5×ATR |
| 追踪止盈 | 关闭 (用策略反转出场) |
| 每笔风险 | $200 (2% of $10k) |
| 年度净盈亏 | **+$1,457.82** |
| 年化收益率 | **+14.6%** |
| 交易次数 | 36笔/年 |
| 胜率 | 30.6% (低胜率高盈亏) |
| 最大回撤 | 6.9% |
| 平均盈利 | +$350 |
| 平均亏损 | -$93 |

### 为什么这个方案赢了？

1. **日线框架**：日均线比4h更稳定，信号质量高，交易次数少
2. **无追踪止盈**：让利润奔跑，直到趋势反转信号出现才出场。这是关键——之前的追踪止盈太早出场，平均每笔只赚$3-11，连佣金都覆盖不了
3. **每笔风险$200**：2%风险比例让收益有意义
4. **30%胜率但高盈亏比**：少数大赚覆盖多次小亏，典型的趋势跟踪盈利模式

## 💻 如何使用

双击桌面 `BTC_Trader.bat` 打开面板：
- **仪表盘**: 实时BTC价格 + 4个策略信号
- **回测**: 选策略跑历史回测
- **实时**: 自动刷新信号
- **全策略对比**: 一键对比4个策略

推荐在 **日线 + RSI+EMA** 上使用，回测选择 "Day" 周期。
